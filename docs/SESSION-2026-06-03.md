## 1. Ce qu'on a accompli aujourd'hui

### Qualité & architecture du plugin
- **Refacto Zustand + useReducer + hooks custom** (`1be3fe8`) — élimination du props drilling, état du DiffScreen organisé, `diffReducer` en fonction pure, `timeAgo` extrait, `opacity` repassé en state local. **60 tests plugin** ajoutés (store ×21, diffReducer ×22, utils ×15).

### Isolation & multi-utilisateurs
- **Storage par branche** (`d1ed9af`) — `{assetId}/{branch}/v{n}.json` au lieu de `{assetId}/v{n}.json` ; évite la collision silencieuse entre `main/v1` et `feature/v1`.
- **Clé de fichier partagée entre designers** (`3c973a8`) — résolution via `figma.root.getPluginData` (stockée dans le fichier, partagée) avant le fallback `clientStorage` (par utilisateur). Deux designers sur le même fichier partagent désormais le même projet.
- **Diff cross-branch** (`3c973a8`) — détection des branches clonées (IDs différents après `clone()`) et matching par chemin d'arbre `TYPE:nom/index`, au lieu de 100 % ajouté/supprimé.

### Sécurité & modèle économique
- **Limite plan Free appliquée côté serveur** (`92c36fc`) — 10 checkpoints/asset vérifiés dans le backend, plus seulement dans l'UI (contournable avant).
- **Cohérence de nœud par branche** (`92c36fc`) — refus si une branche suit déjà un autre élément Figma (évite un diff silencieusement corrompu).
- **Rate limiting** sur `auto-init` (`92c36fc`) — 10 req/min/IP.

### Diff plus précis
- **Tous les fills comparés** (pas seulement le premier) + **effets comparés par valeur** (rayon, couleur, visibilité) et non plus par simple comptage (`92c36fc`).
- **Diff des polices** : `fontFamily`, `fontWeight`, `fontStyle` détectés ; `fontWeight`/`fontStyle` ajoutés au schéma Zod (ils étaient silencieusement supprimés) (`28e8efd`).
- **Mode "Différence"** dans l'overlay (`fa83ef0`) — `mix-blend-mode: difference` : les zones inchangées deviennent noires, les changements s'illuminent. Révèle les changements "sur place" (graisse, couleur) que le fondu ne montre pas.
- **Vrai crossfade** dans le mode Fondu (`82c364d`) — les deux couches s'estompent de façon complémentaire ; "à fond avant" masque vraiment "après".

### Restauration sur canvas (le gros morceau)
- **Cross-branch via `exportAsync` SVG** (`283bb26`, `6e5a315`) — `createNodeFromSvg` avec le SVG capturé, pas de reconstruction ; remplacement par nom + undo atomique.
- **Restore exhaustif, source unique de vérité** (`3ea076c`) — `applyFullSnapshot` réapplique **chaque propriété de chaque nœud** par ID via la fonction unique `applyDeltaProps` : opacité, visibilité, rotation, cornerRadius, strokeWeight, fills (solid + dégradé), strokes, effets, police (famille/graisse/style), texte, taille, vectorPaths, géométrie. Suppression du chemin delta qui ratait des propriétés (ex. la couleur restée pendant que la font était restaurée).
- **Police restaurée à l'identique** (`df176e8`) — `fontStyleName` stocke la chaîne de style Figma brute (`"Medium"`, `"Book Italic"`…) ; round-trip exact au lieu d'une approximation via un poids numérique.
- **`figma.commitUndo()`** (`8d58550`) — toute la restauration = un seul Ctrl+Z.
- **Détection page robuste** (`407c261`) — remonte la chaîne de parents jusqu'à la PAGE (gère les nœuds imbriqués).

## 3. Limites connues (à dire honnêtement)

| Limite | Impact | Note |
|--------|--------|------|
| Checkpoints **antérieurs** aux fix du jour | Pas de `fontStyleName`/`fontWeight` stockés → restore de police impossible dessus | Irréversible pour l'historique ; les nouveaux checkpoints sont corrects |
| Fills **IMAGE** non capturés | Un nœud `[image, solid]` perd son image au restore | Pré-existant, exposé davantage par le full restore |
| Texte **multi-polices** (`figma.mixed`) | Capturé comme `undefined`, restauré en police unique | Cas avancé, hors MVP |
| **Rename** de calque | Casse le matching cross-branch (path = `TYPE:nom/index`) | Affiche supprimé+ajouté au lieu de modifié |
| Nœuds **ajoutés** | Non recréables au restore (reportés comme ignorés) | OK si le canvas est en avance sur la version |
| Mode **Différence** sur SVG transparents | Rendu potentiellement bruité (fond non opaque) | **À valider visuellement** par l'early adopter |
| Restore **"touche tout"** | Repose les propriétés inchangées (no-op) | Non-destructif, mais pas minimal |

## 4. Roadmap V2

### Priorité haute — fiabilité du restore
1. **Restore "live-diff"** — comparer chaque propriété du snapshot à la valeur **actuelle** du nœud sur le canvas, ne poser que les vraies différences. Minimal **et** complet (vs. full-set aveugle actuel). ~30 lignes.
2. **Capture/restore des fills IMAGE** — stocker la référence d'image (`imageHash`) et la réappliquer.
3. **Texte multi-polices** — capture par plage de caractères (`getRangeFontName`) pour les nœuds `figma.mixed`.

### Priorité moyenne — diff visuel
4. **Highlight des nœuds modifiés** — cadre coloré (bounding box) sur les nœuds changés, lu depuis `analysis_json`. Robuste pour tout type de changement, indépendant du rendu.
5. **Mode Différence fiabilisé** — fond opaque forcé pour un rendu propre sur SVG transparents.
6. **Blink comparator** — alternance automatique avant/après (l'œil capte le mouvement).
7. **Matching cross-branch résistant au rename** — heuristique de similarité (géométrie + type) en plus du chemin.

### Priorité produit
8. **SVG pixel-perfect via Figma REST API** — OAuth 2.0, rendu 100 % fidèle (cf. `project_option2_svg`).
9. **Page web pricing + dashboard** (Next.js) — souscription, profil créateur.
10. **Merge entre branches** — la demande n°1 après les branches (relevée par l'early adopter).
11. **Collaboration temps réel** — notifications push (WebSocket), commentaires, approval flow Gold multi-designers.

### Dette technique
12. Nettoyer le champ `delta` (encore envoyé par `ui.tsx`, ignoré par `main.ts` depuis le restore unifié).
13. Pagination de `/api/branches/tree` (renvoie tout l'historique).
14. Contrainte unique DB `(asset_id, branch_name, version_number)`.
